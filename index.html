Responsive Image Markup Patterns

Documenting useful ways to include images in a responsive context.

Examples

Please study the markup in these examples, and be sure to view with JavaScript disabled, too.

Article Listing

An image to illustrate the headline story is considered valuable content. Images for subsequent stories are considered an enhancement, but references to thumbnail images are provided so that these can be shown in certain states.

In this example, thumbnail images for subsequent stories are not shown when a user views the component on a display that is narrower than 640px.

Image Gallery (Slideshow)

As a gallery of images, it’s essential these can be seen by any user viewing this component. A high resolution version of each image (1440×960 pixels) is linked to from a small thumbnail (360×240 pixels).

In this example, when we enhance the design with JavaScript, we use the thumbnails as image placeholders, that appear before the large resolution images have been fully downloaded.

Image Gallery (Carousel)

As a gallery of images, it’s essential these can be seen by any user viewing this component. A high resolution version of each image (1440×960 pixels) is linked to from a small thumbnail (360×240 pixels).

In this example, when we enhance the design with JavaScript, we retain the thumbnail images and include them in the carousel interface. We also use them as placeholder images, appearing before the large resolution images have been fully downloaded.

Big Picture

As a page of large immersive images, it’s essential these can be seen by any user viewing this component. A high resolution version of each image (990 pixels wide) is linked to from a small thumbnail (360 pixels wide).

In this example, when we enhance the design with JavaScript, we use the thumbnails as image placeholders, that appear before the large resolution images have been fully downloaded.
